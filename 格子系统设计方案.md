# 砰砰军团 — 格子系统设计方案

> 本文档定义格子系统的设计方案，供讨论确认后实施。

---

## 一、网格定义

### 布局

```
1 行 × 24 列（水平排列）
列索引: 0  1  2  3  4  5 ... 18 19 20 21 22 23
        |  |  |  |  |  |     |  |  |  |  |  |
```

- 格子从左到右编号 0 ~ 23
- 每个格子在世界空间中有一个固定坐标（中心点）

### 特殊格子（允许占两格）

```
列 2, 3   ← 左防御塔位置（两格宽），可同时容纳 塔 + 1 个其他单位
列 20, 21 ← 右防御塔位置（两格宽），同上
其余所有格子 → 严格 1 个单位
```

## 二、GridPosition — 格子坐标

```csharp
[System.Serializable]
public struct GridPosition
{
    public int col;

    public GridPosition(int col) { this.col = col; }

    // 两格距离（格子为单位）
    public static int Distance(GridPosition a, GridPosition b)
        => Mathf.Abs(a.col - b.col);
}
```

所有单位用 `GridPosition` 定位，不再用自由浮点坐标。

## 三、GridManager — 全局格子管理器

```csharp
public class GridManager : MonoBehaviour
{
    public static GridManager Instance { get; private set; }

    public int Cols = 24;
    public float CellSize;            // 每格宽度（Unity 单位）
    public Vector2 GridOrigin;        // 0 号格子中心的世界坐标

    // 格子坐标 → 世界坐标（用于 UnitVisual 平滑移动）
    public Vector3 GridToWorld(GridPosition pos);

    // 世界坐标 → 格子坐标（用于部署时鼠标点击）
    public GridPosition WorldToGrid(Vector3 worldPos);

    // 格子是否在范围内
    public bool IsInBounds(GridPosition pos)
        => pos.col >= 0 && pos.col < Cols;

    // 格子是否可站立（未被占用，或属于特殊格且满足条件）
    public bool CanOccupy(GridPosition pos, UnitBase unit);
}
```

### 格子占用规则

```csharp
public bool CanOccupy(GridPosition pos, UnitBase unit)
{
    // 越界
    if (!IsInBounds(pos)) return false;

    // 检查当前格子上已有的单位
    var occupants = GetOccupants(pos);
    
    // 空位 → 可放
    if (occupants.Count == 0) return true;

    // 已有人
    if (IsTowerCell(pos))
    {
        // 特殊格：允许 1 塔 + 1 非塔
        bool hasTower = occupants.Any(u => u is Tower);
        bool isTower = unit is Tower;
        
        if (hasTower && isTower) return false;  // 已有塔
        if (occupants.Count >= 2) return false;  // 已满
        return true;
    }

    // 普通格：已有人 → 不可放
    return false;
}

private bool IsTowerCell(GridPosition pos)
    => pos.col == 2 || pos.col == 3 || pos.col == 20 || pos.col == 21;
```

## 四、单位移动 — 格子化

### 移动频率

- 每 **1 小节（8 拍）** 移动一次
- 触发时机：`beat == 1`
- 移动耗时：整小节（视觉在 8 拍内平滑滑动）

### 移动距离

```csharp
// 每小节前进 moveSpeed 格
// moveSpeed = 1 → 每小节走 1 格
// moveSpeed = 2 → 每小节走 2 格
GridPosition newPos = new GridPosition(
    GridPos.col + (int)moveSpeed * direction
);
```

> `moveSpeed` 为整数，小数部分暂不支持（或向下取整 + 累加）

### 移动限制

```
新格子必须在 [0, 23] 范围内
新格子必须无人占用（或满足特殊格规则）
```

### 移动流程

```
OnBeat(bar, 1):
  1. 计算目标格子 = 当前格子 + moveSpeed * 方向
  2. 检测目标格子是否可进入（边界 + 占用）
  3. 可进入 → 更新 GridPosition，UnitVisual 在下一小节内滑过去
  4. 不可进入 → 停在原地
```

### 索敌与攻击

```csharp
InAttackRange(target):
    return Mathf.Abs(GridPos.col - target.GridPos.col) <= attackRange;
```

- `attackRange` 单位为**格子数**
- `attackRange = 1` → 相邻格

## 五、UnitBase 改造要点

```csharp
public class UnitBase : MonoBehaviour
{
    public GridPosition GridPos { get; set; }
    public Vector3 WorldPos => GridManager.Instance.GridToWorld(GridPos);

    // OnBeat:
    //   1. 索敌
    //   2. 在攻击范围内 → 攻击
    //   3. 不在范围内 & beat==1 → 计算目标格子，移动
    //   4. 不可移动 → Idle

    // InAttackRange: 比较 GridPos.col 差值
    // FindNearestEnemy: 遍历单位，找 col 最近的敌方
}
```

### 停止移动判断

- 目标格子被占用 → 停留在原地
- 当移动路径上的格子被阻挡时，停在阻挡前最后一格（未来优化，第一版先停住）
