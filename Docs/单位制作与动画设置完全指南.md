## 一、整体流程

```
精灵图 PNG ─→ 切片 → 创建 Animation Clip → 创建 Animator Controller
                                                      ↓
                  UnitData (SO) ← 预制体 ← 挂载组件 ←
                      ↓
                  部署到场景 → 运行测试
```

## 二、导入精灵图并切片

### 2.1 拖入图片
将序列图 PNG（如 `剑士1_攻击.png`）拖入 `Assets/Objects/`

### 2.2 设为多精灵模式
选中图片，Inspector 设置：
- **Texture Type**: `Sprite (2D and UI)`
- **Sprite Mode**: `Multiple`
- **Pixels Per Unit**: `100`（与项目一致）
- **Filter Mode**: `Point (no filter)`（像素风格）或 `Bilinear`
- **Alpha Is Transparency**: ✅ 勾选
- 点 **Apply**

### 2.3 切片
1. 点 **Sprite Editor** 按钮
2. 点左上角 **Slice**
3. 选 `Grid by Cell Size`，填写每帧宽高
4. 点 **Slice** → 右上角 **Apply**

切完后该图片变成"文件夹"形态，展开能看到所有切出的 Sprite 子资源。

## 三、创建 Animation Clip

### 3.1 准备工作
在 `Assets/` 下创建 `Animations/` 文件夹

### 3.2 创建 Clip
1. 选中切片后的**所有攻击帧**（Shift 多选）
2. 右键 → **Create → Animation**
3. 保存到 `Assets/Animations/`，命名如 `剑士1_Attack.anim`
4. 同理为移动、待机、死亡动画分别创建 Clip

### 3.3 调整帧率
打开 Animation 窗口（Ctrl+6），选中 `.anim` 文件，在时间轴上方将 **Samples** 设为 **8**（一拍播完，对齐节拍）。

### 动画命名规范

| 文件命名 | 对应状态 |
|---------|---------|
| `剑士1_Idle.anim` | 待机 |
| `剑士1_Run.anim` | 移动 |
| `剑士1_Attack.anim` | 攻击 |
| `剑士1_Death.anim` | 死亡 |

## 四、创建 Animator Controller

### 4.1 创建 Controller
`Assets/Animations/` 下右键 → **Create → Animator Controller** → 命名如 `剑士1.controller`

### 4.2 拖入 Clip
1. 双击打开 `.controller`
2. 将上一步创建的所有 `.anim` 文件从 Project 拖入 Animator 窗口

### 4.3 创建参数
Animator 窗口左侧 **Parameters** 选项卡 → 点 **+** → **Int** → 命名 `Action`

### 4.4 设置过渡
右键 **Any State** → **Make Transition** → 点选目标动画状态。对每个过渡箭头，在 Inspector 中设置：

| 过渡 | Conditions | Has Exit Time | Transition Duration |
|------|-----------|---------------|-------------------|
| Any State → 剑士1_Idle | `Action Equals 0` | ❌ 取消 | 0 |
| Any State → 剑士1_Run | `Action Equals 1` | ❌ 取消 | 0 |
| Any State → 剑士1_Attack | `Action Equals 2` | ❌ 取消 | 0 |
| Any State → 剑士1_Death | `Action Equals 3` | ❌ 取消 | 0 |

### Action 值对照表

| 值 | 枚举 | 含义 |
|---|------|------|
| 0 | `Idle` | 待机 |
| 1 | `Moving` | 移动 |
| 2 | `Attacking` | 攻击 |
| 3 | `Dead` | 死亡 |

## 五、创建 UnitData 资产

### 5.1 创建 SO
Project 窗口右键 → **Create → 砰砰军团 → 单位数据** → 命名如 `剑士1_Data`

### 5.2 填写参数

| 字段 | 说明 | 示例值 |
|------|------|--------|
| `unitName` | 单位名称 | 剑士1 |
| `prefab` | 预制体引用（暂空，做 prefab 后再拖入） | — |
| `icon` | UI 图标 | (可选) |
| `unitType` | Melee / Ranged | Melee |
| `maxHP` | 生命值 | 10 |
| `attackPower` | 攻击力 | 2 |
| `attackRange` | 攻击距离 | 1 |
| `attackIntervalInBeats` | 攻击间隔（拍数） | 4 |
| `moveSpeed` | 每拍移动距离 | 0.15 |
| `deployCost` | 部署费用 | 3 |

## 六、创建预制体

### 6.1 搭建预制体
将第一帧精灵（或任意一帧）拖入场景 → 命名如 `剑士1`

### 6.2 添加必要组件

| 组件 | 说明 |
|------|------|
| `SpriteRenderer` | 自动生成，指向拖入的精灵 |
| `Animator` | 新增，Controller 指向 `剑士1.controller` |
| `BoxCollider2D` | (可选，后续碰撞用) |

### 6.3 挂载脚本组件

| 脚本 | 适用单位 |
|------|---------|
| `UnitBase` | 通用单位（需继承扩展） |
| `EnemyBase` | 敌人单位 |
| `Tower` | 防御塔 |

### 6.4 挂载 UnitVisual
- 添加 `UnitVisual` 组件
- 将 `SpriteRenderer` 拖入其引用槽
- `Smooth Speed` 控制平滑移动速度（默认 12）

### 6.5 拖入预制体
从 Hierarchy 拖回 `Assets/Prefabs/`

### 6.6 补全 UnitData 引用
打开 `剑士1_Data`，将预制体拖入 `prefab` 字段

## 七、挂载组件到预制体

最终预制体应包含：

```
剑士1 (GameObject)
├── Transform
├── SpriteRenderer
├── Animator               ← Controller: 剑士1.controller
├── UnitBase (脚本)         ← Data: 剑士1_Data
├── UnitVisual (脚本)       ← SpriteRenderer: 同上
└── BoxCollider2D (可选)
```

> Animator 会自动控制 SpriteRenderer 的 Sprite 属性，原有 Sprite 赋值会被覆盖。

## 八、写单位脚本（可选）

如果现有 `UnitBase` / `EnemyBase` / `Tower` 满足需求，跳过此步。

如需新行为，在 `Assets/Scripts/Units/` 下建新类，继承 `UnitBase`，重写 `OnBeat`：

```csharp
using UnityEngine;

namespace PPCorps
{
    public class MyNewUnit : UnitBase
    {
        public override void OnBeat(int bar, int beat)
        {
            if (_isDead || data == null) return;

            _currentTarget = FindNearestEnemy();

            if (_currentTarget != null && InAttackRange(_currentTarget))
            {
                TryAttack(_currentTarget);
            }
            else
            {
                SetAction(UnitAction.Idle);
            }
        }
    }
}
```

**重要**：必须使用 `SetAction(UnitAction.xxx)` 而非直接赋值 `_currentAction`，否则动画不会同步。

## 九、场景挂载与测试

### 9.1 将预制体放入场景
从 Project 拖 `剑士1.prefab` 到场景（或让 DeploySystem 运行时实例化）

### 9.2 运行测试
- 单位移动时 → Animator 切换到 Run 动画
- 单位攻击时 → Animator 切换到 Attack 动画
- 单位死亡时 → Animator 切换到 Death 动画

### 9.3 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 精灵不显示 | Animator 未接管 SpriteRenderer | 检查预制体上 SpriteRenderer 是否存在 |
| 动画不切换 | `Action` 参数名不一致 | 代码中 `SetInteger("Action", ...)` 与 Animator 参数名必须一致 |
| 动画一闪而过 | Samples 太高 | 设为 8~12 |
| Animator 参数没更新 | `_animator` 为 null | 检查 `[SerializeField]` 槽是否拖入了 Animator |

## 十、快速参考卡

```
1. 拖 PNG → Sprite Mode: Multiple → Sprite Editor 切片 → Apply
2. 选中帧 → 右键 Create Animation → 命名保存
3. 设 Samples = 8
4. 创建 Animator Controller → 拖入所有 Clip
5. 添加 Int 参数 "Action"
6. Any State → 每个 Clip，条件 Action=0/1/2/3，Duration=0
7. 创建 UnitData (SO) → Create → 砰砰军团 → 单位数据
8. 拖精灵到场景 → 加组件（Animator + 脚本 + UnitVisual）
9. 拖回 Project 做成 Prefab
10. SO 的 prefab 字段指向预制体
11. UnitData.attackRange > 0.5 即为远程攻击
```
