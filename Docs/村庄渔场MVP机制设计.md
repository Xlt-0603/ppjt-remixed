# 砰砰军团 — 渔场 MVP 玩法机制设计

> 目标：以最小工作量实现可玩的钓鱼玩法，后续迭代补齐完整机制。
> 对应完整机制文档：`原始砰砰军团完整玩法设计.md` 第五节。

---

## 一、MVP 范围取舍

| 机制 | 完整版 | MVP | 理由 |
|------|--------|-----|------|
| 钓鱼区域 | 5 个 | **1 个**（林间小溪） | 多区域只是换皮，核心机制不变 |
| 钓鱼模式 | 手动 + 自动 | **仅自动** | 手动节奏音游开发量大，后续加 |
| 鱼竿 | 6 把（含长度区间） | **1 把**（可升级等级） | 用等级代替多竿，简化系统 |
| 鱼钩 | 5+ 种（特质匹配） | **取消** | 特质匹配太复杂，MVP 跳过 |
| 鱼饵 | 5 种 | **保留 2 种**：普通饵 / 高级饵 | 有选择感但复杂度低 |
| 辅助设备 | 鱼箱 + 小猫 | **仅鱼箱**（容量升级） | 小猫加速度是纯数值，可后续加 |
| 鱼的属性 | 名称/体型/特质/金冠银冠 | **名称 + 体型 + 稀有度** | 简化为蓝/紫/金三档 |
| 概率公式 | 4 层过滤 | **1 层**（体型权重 + 稀有度权重） | 一刀切，好理解好实现 |
| 图鉴系统 | 每条鱼金冠银冠 | **收集点亮 + 完成度百分比** | 仅记录钓到与否 |

## 二、MVP 完整流程

```
1. 玩家点击渔场建筑 → 打开渔场面板
2. 面板显示：当前鱼币、装备槽、开始钓鱼按钮、图鉴入口
3. 点击"开始钓鱼" → 进入自动钓鱼模式
4. 钓鱼过程中显示进度条，自动缓慢增长
5. 进度条满 → 结算一条鱼（弹出鱼获弹窗）
6. 鱼自动收入图鉴（首次点亮）并获得鱼币
7. 积累鱼币 → 升级鱼竿 / 升级鱼箱 / 购买鱼饵
```

|------|------|--------|---------|---------|
| 溪流鲫鱼 | 小 | 普通（蓝） | 40% | 5 |
| 石斑鱼 | 小 | 普通（蓝） | 25% | 8 |
| 赤眉鱼 | 中 | 稀有（紫） | 15% | 15 |
| 荧光水母 | 中 | 稀有（紫） | 10% | 20 |
| 火焰泥鳅 | 中 | 稀有（紫） | 5% | 30 |
| 金鳞鲤 | 大 | 传说（金） | 3% | 50 |
| 暗影鲶 | 大 | 传说（金） | 1.5% | 80 |

### 3.3 自动钓鱼逻辑

```
每 tick（约 3 秒/次）：
  1. 鱼箱已存数量 +1
  2. 按当前概率表抽取一条鱼
  3. 存入鱼箱（暂不展示）
  4. 鱼箱满 或 玩家手动点击"收竿" → 批量结算

结算时：
  - 遍历鱼箱中所有鱼
  - 首次获得的鱼 → 图鉴点亮 + 弹出"NEW!"提示
  - 已有鱼 → 仅累加鱼币
  - 展示结算弹窗：总鱼币 + 新鱼列表
```

### 3.4 概率抽取算法

```csharp
float GetFishWeight(FishData fish, bool useBait, int rodLevel)
{
    float w = fish.baseWeight;
    if (useBait && fish.rarity >= Rarity.Rare)
        w *= 2;
    if (fish.size == FishSize.Large)
        w += rodLevel * 0.01f;
    return w;
}

FishData RollFish(bool useBait, int rodLevel)
{
    float total = 0;
    foreach (var fish in allFishes)
        total += GetFishWeight(fish, useBait, rodLevel);

    float roll = Random.Range(0, total);
    float cumulative = 0;
    foreach (var fish in allFishes)
    {
        cumulative += GetFishWeight(fish, useBait, rodLevel);
        if (roll <= cumulative)
            return fish;
    }
    return allFishes[0];
}
```
