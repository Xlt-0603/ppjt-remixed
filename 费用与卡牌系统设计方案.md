## 二、费用系统 (EnergyManager)

### 2.1 核心参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `maxEnergy` | 10 | 费用上限 |
| `currentEnergy` | 3 | 当前可用费用 |
| `regenPerBar` | 2 | 每小节回复费用数 |
| `startEnergy` | 3 | 部署阶段初始费用 |

### 2.2 回复机制（节拍绑定）

费用回复与节拍系统挂钩：

```
每小节 (Bar) 开始时回复 regenPerBar 点费用
  ├─ Bar 1 开始 → +2 (cap 10)
  ├─ Bar 2 开始 → +2 (cap 10)
  └─ ...
```

在 `GameManager.OnBeat(bar, beat)` 中，当 `beat == 1` 时触发回复：

```csharp
// GameManager 中
public void OnBeat(int bar, int beat)
{
    if (beat == 1 && State == GameState.Deploy)
        EnergyManager.Instance.AddEnergy(EnergyManager.Instance.regenPerBar);
}
```

### 2.3 关键 API

```csharp
public class EnergyManager : MonoBehaviour
{
    public static EnergyManager Instance { get; private set; }

    [SerializeField] private int _maxEnergy = 10;
    [SerializeField] private int _startEnergy = 3;
    [SerializeField] private int _regenPerBar = 2;

    public int CurrentEnergy { get; private set; }
    public int MaxEnergy => _maxEnergy;
    public int RegenPerBar => _regenPerBar;

    public event Action<int, int> OnEnergyChanged;  // (current, max)

    // 消耗费用，成功返回 true
    public bool Spend(int amount)
    {
        if (amount > CurrentEnergy) return false;
        CurrentEnergy -= amount;
        OnEnergyChanged?.Invoke(CurrentEnergy, _maxEnergy);
        return true;
    }

    // 增加费用（不超过上限）
    public void AddEnergy(int amount)
    {
        CurrentEnergy = Mathf.Min(_maxEnergy, CurrentEnergy + amount);
        OnEnergyChanged?.Invoke(CurrentEnergy, _maxEnergy);
    }

    // 战场阶段切换时冻结回复
    public void OnStateChanged(GameState newState)
    {
        if (newState == GameState.Battle)
            // 进入战斗后停止回复，保留剩余费用
    }
}
```

### 2.4 费用消耗流程

```
玩家拖拽卡牌到网格
  → CardManager 检查费用: EnergyManager.Spend(cost)
  → 费用足够 → 扣费 → 单位生成
  → 费用不足 → UI 反馈（闪烁红色/震动）
```

## 三、卡牌系统 (CardManager)

### 3.1 卡组 (Deck) 配置

玩家拥有一个卡组（Deck），包含游戏中所有可用单位及其数量：

```csharp
[System.Serializable]
public class CardEntry
{
    public UnitData unitData;      // 引用单位数据 SO
    public int count = 1;          // 卡组中剩余数量
}

public class CardManager : MonoBehaviour
{
    public static CardManager Instance { get; private set; }

    [SerializeField] private List<CardEntry> _deck;        // 初始卡组
    [SerializeField] private int _handSize = 4;             // 手牌上限
    [SerializeField] private int _drawPerBar = 1;           // 每小节抽牌数

    private List<UnitData> _hand = new List<UnitData>();   // 当前手牌
    private Queue<UnitData> _drawPile = new Queue<UnitData>(); // 抽牌堆
}
```

### 3.2 抽牌机制

节拍驱动，与费用回复同步：

```
每小节 (Bar) 开始时：
  1. 回复费用（EnergyManager）
  2. 抽 _drawPerBar 张牌（手牌不满时）
```

### 3.3 关键 API

```csharp
public class CardManager : MonoBehaviour
{
    // 手牌（只读）
    public IReadOnlyList<UnitData> Hand => _hand.AsReadOnly();
    // 手牌变化事件（UI 刷新）
    public event Action OnHandChanged;

    // 初始化：构建抽牌堆（洗牌）
    private void BuildDrawPile()
    {
        _drawPile.Clear();
        foreach (var entry in _deck)
        {
            for (int i = 0; i < entry.count; i++)
                _drawPile.Enqueue(entry.unitData);
        }
        Shuffle(_drawPile);
    }

    // 抽一张牌（手牌不满时）
    public void DrawOne()
    {
        if (_hand.Count >= _handSize) return;
        if (_drawPile.Count == 0) return;
        _hand.Add(_drawPile.Dequeue());
        OnHandChanged?.Invoke();
    }

    // 使用一张卡（拖拽放置后调用）
    public void UseCard(int handIndex)
    {
        _hand.RemoveAt(handIndex);
        OnHandChanged?.Invoke();
        // 抽一张新牌补位
        DrawOne();
    }

    // 替换一张卡（消耗费用）
    public bool ReplaceCard(int handIndex, int cost = 1)
    {
        if (!EnergyManager.Instance.Spend(cost)) return false;
        _hand.RemoveAt(handIndex);
        DrawOne();
        return true;
    }
}
```

## 四、UI 设计方案

### 4.1 UI 层级结构

```
UI (Canvas) — Screen Space - Overlay
├── EnergyBar (Panel)          ← 底部费用条

├── CardHand (Panel)           ← 底部卡牌区（费用条上方）

│
├── CardDragPreview (Image)    ← 拖拽时的跟随图标
│
```

### 4.2 卡牌卡片 (CardWidget) 设计

```
┌──────────┐
│ ⚔        │  ← 单位类型图标（近战/远程）
│          │
│  [icon]  │  ← UnitData.icon
│          │
│  剑士1   │  ← 单位名称
│  cost 3  │  ← 费用（数字 + ⚡标记）
└──────────┘
```

卡片状态：
- **正常**：费用足够时正常显示
- **费用不足**：灰色遮罩 + 红色费用数字
- **被选中**：放大 + 高亮边框
- **拖拽中**：原位置半透明，鼠标跟随副本

### 4.3 费用条 (EnergyBar) 设计

```
┌─────────────────────────────────┐
│ ⚡ 8/10  [████████░░░░░░░░░]   │
└─────────────────────────────────┘
  ↑ 图标    ↑ 数字    ↑ 进度条（绿色）
```

- 回复动画：数字跳动 + 进度条平滑过渡
- 费用不足时闪烁红色
- 颜色变化：绿色（充足）→ 黄色（< 3）→ 红色（不够花费）

### 4.4 拖拽交互流程

```
1. 手指/鼠标按下卡牌
   → 创建拖拽副本 (CardDragPreview)
   → 原卡片半透明
   → 记录起始位置

2. 拖拽到战场区域
   → 鼠标位置 → WorldToGrid → 获取目标列
   → 显示绿色预览框（可放置）
   → 或红色预览框（不可放置：费用不足/格子被占）

3. 松手
   ├─ 放置合法 → 扣费 → 移除手牌 → 抽新牌 → 生成单位
   └─ 放置非法 → 拖拽副本飞回原位 → 取消操作
```

### 4.5 放置合法性检查

```csharp
public bool CanDeploy(UnitData unitData, GridPosition pos)
{
    // 1. 费用检查
    if (EnergyManager.Instance.CurrentEnergy < unitData.deployCost)
        return false;
    // 2. 格子上限检查（Tower 可以和单位共存）
    if (!GridManager.Instance.CanOccupy(pos, null))
        return false;
    // 3. 边界检查
    if (!GridManager.Instance.IsInBounds(pos))
        return false;
    // 4. 阵营区域检查（玩家只能放在我方半场）
    if (!IsPlayerSide(pos))
        return false;
    return true;
}
```

### 4.6 阵营区域划分

```
┌──────────────────────────────────────────────────┐
│  敌人区域 (cols 12-23)     │  玩家区域 (cols 0-11) │
│  ← 只能放敌人单位         │  → 只能放玩家单位     │
│                            │                       │
│  敌人卡组                  │  玩家卡组             │
│  敌人费用系统              │  玩家费用系统         │
└──────────────────────────────────────────────────┘
```

玩家放置区：列 0-11（左侧半场）
敌人放置区：列 12-23（右侧半场，可选 AI 或手动）

## 五、部署系统 (DeploySystem)

### 5.1 职责

```csharp
public class DeploySystem : MonoBehaviour
{
    public static DeploySystem Instance { get; private set; }

    // 在指定网格列生成单位
    public GameObject DeployUnit(UnitData data, GridPosition pos, bool isEnemy = false)
    {
        // 1. 检查合法性
        if (!CanDeploy(data, pos)) return null;

        // 2. 消耗费用
        EnergyManager.Instance.Spend(data.deployCost);

        // 3. 实例化预制体
        Vector3 worldPos = new Vector3(
            GridManager.Instance.GridToWorldX(pos),
            0, -3
        );
        GameObject go = Instantiate(data.prefab, worldPos, Quaternion.identity);

        // 4. 初始化单位（设置阵营、网格位置）
        UnitBase unit = go.GetComponent<UnitBase>();
        unit.Init(pos, isEnemy);

        return go;
    }
}
```

### 5.2 部署模式切换

```
Deploy 阶段：
  ├─ 玩家自由拖拽放置
  ├─ 费用每小节回复
  ├─ 手牌用完后自动抽取
  └─ 点击"开战" → Battle 阶段

Battle 阶段：
  ├─ 禁用拖拽放置
  ├─ 费用冻结（不再回复）
  └─ 自动战斗
```

## 七、与现有系统的关系

```
GameManager
├─ EnergyManager   ← 新增，由 GameManager 驱动费用回复
├─ CardManager     ← 新增，由 GameManager 驱动抽牌
├─ DeploySystem    ← 新增，拖拽入口
├─ GridManager     ← 已有，DeploySystem 使用其网格验证
├─ MetronomeUI     ← 已有，调试面板保持不变
├─ UnitBase        ← 已有，DeploySystem 使用其生成单位
└─ UnitData        ← 已有，含 deployCost、icon、prefab 字段
```

### 数据流

```
OnBeat(bar, 1) [Deploy 阶段]
  ├─ EnergyManager.AddEnergy(regenPerBar)  → 回复费用
  ├─ CardManager.DrawOne()                 → 抽牌
  └─ UI_EnergyBar 更新  +  UI_CardHand 更新

玩家拖拽卡牌到网格
  ├─ CardManager 获取卡牌 UnitData
  ├─ DeploySystem.CanDeploy() 检查合法性
  ├─ EnergyManager.Spend(cost) 扣费
  ├─ DeploySystem.DeployUnit() 生成单位
  ├─ CardManager.UseCard(index) 消耗手牌 + 抽新牌
  └─ UI 全部刷新
```

## 八、后续扩展可能

1. **费用加速**：某些单位/技能可以让费用回复翻倍
2. **卡牌升级**：同一单位的卡牌可以升级（增加 deployCost，提升属性）
3. **特殊卡**：法术牌（不部署单位，触发一次性效果）
4. **卡牌冷却**：使用后进入冷却，冷却结束后回到抽牌堆
5. **多费用类型**：分为"部署费"和"技能费"两种资源
6. **敌人费用系统**：AI 使用独立的费用系统，用于波次生成
