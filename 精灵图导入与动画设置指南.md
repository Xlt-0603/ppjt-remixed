## 一、准备工作

### 你需要准备

- 攻击动画的序列图 PNG（如 `豆丁_攻击.png`，多帧横向排列）
- 移动动画的序列图 PNG（如 `豆丁_移动.png`，多帧横向排列）

### 序列图规格建议

| 项目 | 建议 |
|------|------|
| 每帧大小 | 正方形（如 64×64、128×128），方便对齐网格 |
| 排列方式 | 横向单行排列 |
| 帧数 | 移动 4~6 帧、攻击 2~4 帧 |
| 背景 | 透明背景 |
| Pixels Per Unit | 与现有精灵一致（当前 `剑士1.png` 为 100） |

## 二、导入精灵图并切片

### 2.1 导入图片

将序列图 PNG 拖入 `Assets/Objects/` 目录（与其他精灵图放在一起）。

### 2.2 设置为多精灵模式

1. 在 Project 窗口选中导入的 PNG
2. Inspector 中设置：
   - **Texture Type**: Sprite (2D and UI)
   - **Sprite Mode**: `Multiple` ← 关键！单图多帧必须选这个
   - **Pixels Per Unit**: 与项目一致（现有资源为 100）
   - **Filter Mode**: Point (no filter)（像素风格推荐）或 Bilinear
   - **Alpha Is Transparency**: ✅ 勾选
3. 点击底部的 **Apply**

### 2.3 使用 Sprite Editor 切片

1. 点击 Inspector 中的 **Sprite Editor** 按钮
2. 在弹出的窗口中点击左上角的 **Slice**
3. 选择切片方式（二选一）：

   **方式 A：Grid by Cell Size（推荐）**
   ```
   Slice → Type: Grid by Cell Size
   - Pixel Size: X = 每帧宽度, Y = 每帧高度
   - Offset: X = 0, Y = 0
   - Padding: X = 0, Y = 0
   ```
   点击 **Slice**，会自动按网格切好。

   **方式 B：Automatic**
   适用于各帧之间有明确间距的图，Unity 自动识别边界。

4. 检查切片结果，每个切片应该有正确的矩形框
5. 可双击每个切片重命名（如 `豆丁_移动_0`, `豆丁_移动_1`...）
6. 点击右上角 **Apply**

> **切完后的样子**：原本的 `剑士1.png` 变成一个"文件夹"形态，展开后能看到所有切出的 Sprite 子资源。

## 三、创建 Animation Clip

> Animation Clip 是 Unity 中存储关键帧动画数据的文件，每个 Clip 对应一个完整的动画（如"播放攻击动作"）。

### 3.1 创建文件夹

在 Project 窗口的 `Assets/` 下右键 → Create → Folder，命名为 `Animations`，用于存放所有动画相关文件。

### 3.2 创建 Animation Clip

**方法一：直接从 Sprite 创建（推荐）**

1. 选中切片后的所有攻击帧 Sprite（按住 Shift 点选或框选）
2. 右键 → **Create → Animation**
3. 保存到 `Assets/Animations/`，命名为 `豆丁_攻击.anim`
4. 同理选中移动帧 → 右键 → Create → Animation → 命名为 `豆丁_移动.anim`

**方法二：从 Animation 窗口创建**

1. 选中要挂动画的预制体（或在场景中选中一个实例）
2. 打开 Window → Animation → Animation（或 Ctrl+6）
3. 点击 **Create** 按钮，保存为 `豆丁_攻击.anim`
4. 会自动创建一个 Animator Controller 和第一个 Clip
5. 将攻击帧从 Project 拖入 Animation 窗口的时间轴
6. 同理创建 `豆丁_移动.anim`

### 3.3 调整帧率

在 Animation 窗口中找到 **Samples**（采样率）输入框：

```
Samples: 8
```

> **为什么是 8**？因为游戏以拍为节奏单位（Beat 1~8）。Samples 设为 8 意味着整个动画在 1 拍内播完（约 0.125s@BPM=60），节奏完全对齐。如果想让动画更平滑，可选 12 或 16，但务必实测视觉效果。

调整方法：
1. 打开 Animation 窗口（Ctrl+6）
2. 选中创建的 `.anim` 文件
3. 在时间轴上方找到 Samples 输入框，设置为 **8**
4. 拖动各帧的时间位置微调节奏

## 四、创建 Animator Controller

> Animator Controller 是"动画状态机"，控制什么条件下播放哪个动画。

### 4.1 创建 Controller

1. 在 `Assets/Animations/` 文件夹下
2. 右键 → **Create → Animator Controller**
3. 命名为 `豆丁.controller`

### 4.2 拖入 Animation Clip

1. 双击打开 `豆丁.controller`
2. 将 `豆丁_移动.anim` 和 `豆丁_攻击.anim` 从 Project 拖入 Animator 窗口

### 4.3 创建控制参数

1. 在 Animator 窗口左侧点击 **Parameters** 选项卡
2. 点击 **+** → **Int** → 命名为 `Action`
3. 确保 Parameter 类型是 **Int**

### 4.4 设置状态过渡

状态过渡决定了单位从移动到攻击或反过来时，动画如何切换。

**从 Any State 过渡（最简单推荐）**

1. 右键 Animator 空白处 → 选 `Add State → From Entry` 中的 **Make Transition**，或者右键 Any State → **Make Transition**，连线到 `豆丁_移动`
2. 同理，Any State → `豆丁_攻击`
3. 单击每条过渡箭头，在 Inspector 中设置：

   ```
   Conditions: Action Equals 1   ← 对应移动 (Moving)
   取消勾选 Has Exit Time
   Transition Duration: 0
   ```

   ```
   Conditions: Action Equals 2   ← 对应攻击 (Attacking)
   取消勾选 Has Exit Time
   Transition Duration: 0
   ```

**最终 Animator 结构**：

```
               ┌──────────────────────┐
               │      Any State       │
               └──┬───────────────┬───┘
        Action=1  │               │  Action=2
          ┌───────┘               └────────┐
          ↓                                 ↓
   ┌──────────────┐               ┌──────────────┐
   │ 豆丁_移动.anim │               │ 豆丁_攻击.anim │
   └──────────────┘               └──────────────┘
```

### 参数值对照表

| Action 值 | 枚举 | 含义 |
|-----------|------|------|
| 0 | `Idle` | 待机（无动画） |
| 1 | `Moving` | 播放移动动画 |
| 2 | `Attacking` | 播放攻击动画 |
| 3 | `Dead` | 死亡（可加死亡动画） |

## 五、挂载到预制体

### 5.1 打开预制体

1. 在 Project 窗口中找到 `Assets/Prefabs/剑士1.prefab`
2. 双击打开预制体编辑模式

### 5.2 添加 Animator 组件

1. 选中预制体的根节点
2. 在 Inspector 底部点击 **Add Component**
3. 搜索 **Animator** 并添加
4. 将 **Controller** 指向刚才创建的 `豆丁.controller`
5. 保存预制体（Ctrl+S）

### 5.3 检查已有组件

预制体最终应有这些组件：

```
剑士1 (GameObject)
├── Transform
├── SpriteRenderer      ← Animator 会接管它的 sprite
├── Animator            ← 新增，Controller = 豆丁.controller
├── UnitBase (脚本)     ← 驱动动画切换
└── BoxCollider2D       ← 可能有，不影响
```

> **注意**：Animator 会自动控制 SpriteRenderer 的 Sprite 属性，原有 Sprite 赋值会被覆盖。不需要在 SpriteRenderer 上手动指定图片。

## 六、代码驱动动画

需要在 `UnitBase.cs` 中补充 Animator 的同步逻辑。

### 6.1 添加 Animator 引用

在 `UnitBase.cs` 的现有字段区域添加：

```csharp
[SerializeField] private Animator _animator;
```

### 6.2 在 Start 中获取组件

在现有的 `Start()` 方法中补充获取 Animator 的代码。找到下面这行：

```csharp
if (GetComponent<UnitHPBar>() == null)
    gameObject.AddComponent<UnitHPBar>();
```

在其后添加：

```csharp
if (_animator == null)
    _animator = GetComponent<Animator>();
```

### 6.3 在 OnBeat 末尾同步动画

在 `OnBeat()` 方法的末尾（所有逻辑判断之后），添加：

```csharp
public virtual void OnBeat(int bar, int beat)
{
    if (_isDead || data == null) return;

    _currentTarget = FindNearestEnemy();

    if (_currentTarget != null && InAttackRange(_currentTarget))
    {
        TryAttack(_currentTarget);
    }
    else if (beat == 1)
    {
        if (_currentTarget != null)
            MoveOneStepTowards(_currentTarget.transform.position);
        else
        {
            transform.position += (Vector3)defaultMoveDirection * data.moveSpeed;
            _currentAction = UnitAction.Moving;
        }
    }
    else
    {
        _currentAction = UnitAction.Idle;
    }

    // ==== 新增：同步 Animator ====
    if (_animator != null && _animator.isActiveAndEnabled)
        _animator.SetInteger("Action", (int)_currentAction);
}
```

### 6.4 完整修改预览

`UnitBase.cs` 的完整改动只涉及三处：

| 位置 | 改动 |
|------|------|
| 字段区域 | 添加一行 `[SerializeField] private Animator _animator;` |
| `Start()` | 在获取 UnitHPBar 后加一行获取 Animator 的代码 |
| `OnBeat()` 末尾 | 加几行同步 `_animator.SetInteger("Action", ...)` |

> **注意**：`_currentAction` 在 `TryAttack()` 和 `MoveOneStepTowards()` 方法中已经被正确设置为 `UnitAction.Attacking` 和 `UnitAction.Moving`，所以 Animator 会同步切换。

## 七、验证与调试

### 7.1 在场景中测试

1. 将 `剑士1.prefab` 拖入场景
2. 运行游戏
3. 观察单位在移动时是否切换到移动动画
4. 进入攻击范围时是否切换到攻击动画

### 7.2 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 精灵不显示 | Animator 未正确引用 SpriteRenderer | 检查预制体上是否有 SpriteRenderer |
| 动画不切换 | `Action` 参数名不一致 | 检查代码中的参数名与 Animator 中是否一致 |
| 动画一闪而过 | Samples 数值过高 | 降低 Samples，设为 8~12 |
| 动画不播放 | 过渡条件未设置 | 检查 Any State → Clip 的 Conditions |
| 移动时仍显示攻击帧 | 过渡未断开 | 设置 Transition Duration = 0，取消 Has Exit Time |
| Animator 参数未更新 | `_animator` 为 null | 检查 `_animator` 是否在 Awake/Start 中成功获取 |

### 7.3 节拍对齐检查

在 BPM=60 的情况下：
- 每拍时长 = 0.125s
- Samples=8 → 8帧在 1 拍内播完，每帧约 0.0156s
- 如果移动动画 4 帧，可在 Samples=4 时 1 拍播完，或 Samples=8 时 2 拍播完

根据需要调节 Samples，让动画速度看起来自然。

## 八、进阶技巧

### 8.1 Idle（待机）动画

如果还需要待机动画：
1. 创建 `豆丁_Idle.anim`
2. 在 Animator 中添加新状态
3. 设置条件 `Action Equals 0`
4. `_currentAction` 在无法攻击且非移动拍时会自动设为 `Idle`

### 8.2 为敌人添加动画

对 `剑士enemy.prefab` 重复上述全部步骤，创建敌人专属的精灵图和 Controller。

### 8.3 死亡动画

在 `Die()` 方法中：

```csharp
protected virtual void Die()
{
    _isDead = true;
    _currentAction = UnitAction.Dead;
    if (_animator != null)
        _animator.SetInteger("Action", (int)UnitAction.Dead);

    // 可选：延迟销毁，让死亡动画播完
    // Destroy(gameObject, 0.5f);

    if (GameManager.Instance != null)
        GameManager.Instance.UnregisterUnit(this);

    Destroy(gameObject);
}
```

需要添加死亡动画 Clip，并在 Animator 中添加 `Action=3` 的过渡。

### 8.4 利用 Animation Events

可在攻击动画的某一帧添加 Animation Event，让伤害实际命中时间与动画匹配：

1. 打开 `豆丁_攻击.anim`
2. 在时间轴上右键 → **Add Animation Event**
3. 在 Inspector 中填写函数名 `OnAttackHit`

```csharp
// 在 UnitBase 中添加
public void OnAttackHit()
{
    // 实际造成伤害的逻辑（从 TryAttack 中分离出来）
    if (_currentTarget != null && !_currentTarget.IsDead)
        _currentTarget.TakeDamage(data.attackPower);
}
```
