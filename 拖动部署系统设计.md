## 一、功能概述

```
玩家点击面板上的单位图标
  → 鼠标位置出现半透明"幽灵"单位跟随移动
  → 鼠标移入战场区域时，幽灵吸附到最近的网格点
  → 松开鼠标：
      ├─ 位置有效（在场地内、无人占用、能量够）→ 实例化单位，扣能量
      └─ 位置无效 → 取消部署，幽灵消失
```

### 关键交互细节

- 只有 `GameState.Deploy` 状态下可部署
- 拖动时单位跟随鼠标，但仅吸附到**水平 X 轴网格点**（Y 轴固定在部署行的 Y 坐标）
- 幽灵用半透明 Sprite 显示，绿色 = 可放置，红色 = 不可放置
- 释放后切换到下一单位继续部署

## 三、EnergyManager — 能量管理

### 设计

```
初始能量: 10
自动回复: 每 2 秒 +1
上限: 20
消耗: 部署单位时扣 UnitData.deployCost
```

### 代码设计

```csharp
public class EnergyManager : MonoBehaviour
{
    [SerializeField] private int _maxEnergy = 20;
    [SerializeField] private int _initialEnergy = 10;
    [SerializeField] private float _regenInterval = 2f;  // 秒

    public int CurrentEnergy { get; private set; }

    public event Action<int, int> OnEnergyChanged;  // (current, max)

    // 是否能支付
    public bool CanAfford(int cost) => CurrentEnergy >= cost;

    // 尝试花费，返回是否成功
    public bool Spend(int cost);

    // 每回复一次加 1
    private void StartRegen();
}
```

### 与 UI 对接

`DeployPanel` 监听 `OnEnergyChanged`，更新顶部的能量文字/进度条。

## 四、GridHelper — 网格工具

### 设计

将鼠标屏幕坐标 → 世界坐标 → 对齐到网格点。

### 代码设计

```csharp
public class GridHelper : MonoBehaviour
{
    [SerializeField] private float _tileSize = 1.17f;
    [SerializeField] private int _gridWidth = 10;          // 水平格数
    [SerializeField] private int _gridHeight = 3;           // 垂直格数
    [SerializeField] private Vector2 _gridOrigin;           // 网格左下角世界坐标

    // 鼠标位置 → 最近的网格点世界坐标
    public Vector3 ScreenToGridPoint(Vector2 screenPos);

    // 网格坐标 → 世界坐标
    public Vector3 GridToWorld(int col, int row);

    // 世界坐标 → 网格坐标 (col, row)
    public Vector2Int WorldToGrid(Vector3 worldPos);

    // 判断网格点是否在场地内
    public bool IsInBounds(int col, int row);

    // 判断网格点是否已被单位占据
    public bool IsOccupied(Vector3 gridPos);
}
```

### 网格对齐规则

- **X 轴**：`Mathf.RoundToInt((worldPos.x - origin.x) / tileSize) * tileSize + origin.x`
- **Y 轴**：固定部署行的 Y 坐标（不需要玩家调整 Y）

## 六、DragHandler — 拖拽逻辑

### 设计

```
点击卡片 → 生成幽灵单位 → 跟随鼠标 → 吸附网格 → 释放 → 部署/取消

状态机：
  Idle → Dragging → (Placed / Cancelled) → Idle
```

### 代码设计

```csharp
public class DragHandler : MonoBehaviour
{
    [SerializeField] private GameObject _ghostPrefab;  // 幽灵预制体（半透明）

    private UnitData _currentData;     // 当前拖拽的单位数据
    private GameObject _ghost;         // 当前显示的幽灵实例
    private bool _isDragging;

    private void Update()
    {
        if (!_isDragging) return;

        // 鼠标移动到最近的网格点
        Vector3 gridPos = GridHelper.ScreenToGridPoint(Input.mousePosition);

        // 更新幽灵位置
        _ghost.transform.position = gridPos;

        // 检查是否可以放置 → 切换幽灵颜色
        bool canPlace = ValidatePlacement(gridPos);
        SetGhostColor(canPlace ? Color.green : Color.red);

        // 松开鼠标
        if (Input.GetMouseButtonUp(0))
        {
            if (canPlace)
                DeploySystem.PlaceUnit(_currentData, gridPos);
            CancelDrag();
        }

        // 右键取消
        if (Input.GetMouseButtonDown(1))
            CancelDrag();
    }

    public void StartDrag(UnitData data)
    {
        if (_isDragging) return;
        if (!EnergyManager.Instance.CanAfford(data.deployCost)) return;

        _currentData = data;
        _ghost = Instantiate(_ghostPrefab);
        _ghost.GetComponent<SpriteRenderer>().sprite = data.icon;
        _isDragging = true;
    }

    private void CancelDrag()
    {
        Destroy(_ghost);
        _isDragging = false;
        _currentData = null;
    }

    private bool ValidatePlacement(Vector3 gridPos)
    {
        // 条件：场地内 + 未被占据 + 能量够
        return GridHelper.IsInBounds(gridPos)
            && !GridHelper.IsOccupied(gridPos)
            && EnergyManager.Instance.CanAfford(_currentData.deployCost);
    }
}
```

### 幽灵预制体

一个简单 GameObject，包含：
- `SpriteRenderer`（半透明材质）
- 与单位预制体相同大小的碰撞体（方便视觉参考）
- 无脚本，只做视觉跟随

## 七、DeploySystem — 部署总控

### 代码设计

```csharp
public class DeploySystem : MonoBehaviour
{
    public static DeploySystem Instance { get; private set; }

    [SerializeField] private Transform _unitContainer;  // 部署单位的父级

    private void Awake() => Instance = this;

    // 在指定位置部署单位
    public void PlaceUnit(UnitData data, Vector3 position)
    {
        if (data == null || data.prefab == null) return;

        // 扣能量
        EnergyManager.Instance.Spend(data.deployCost);

        // 实例化
        GameObject unitObj = Instantiate(data.prefab, position, Quaternion.identity);
        unitObj.transform.SetParent(_unitContainer);

        // 初始化（UnitBase 的 Start 会自动注册到 GameManager）
    }
}
```

## 八、与现有系统的对接

### 8.1 状态限制

```
GameManager.State == Deploy 时才能拖拽
进入 Battle 后禁用部署面板
```

在 `DragHandler.Update()` 开头判断：
```csharp
if (GameManager.Instance.State != GameState.Deploy)
{
    if (_isDragging) CancelDrag();
    return;
}
```

### 8.2 单位注册

`UnitBase.Start()` 已自动调用 `GameManager.Instance.RegisterUnit(this)`，无需额外处理。

### 8.3 已占用格子

`GridHelper.IsOccupied()` 需要查询 `GameManager.GetAllUnits()` 来判断格子上是否已有单位。

```csharp
public bool IsOccupied(Vector3 gridPos)
{
    foreach (var unit in GameManager.Instance.GetAllUnits())
    {
        if (unit == null || unit.IsDead) continue;
        if (Vector3.Distance(unit.LogicalPosition, gridPos) < _tileSize * 0.5f)
            return true;
    }
    return false;
}
```

### 8.4 能量回复使用节拍

可选：能量回复不按 realtime 秒，而是按节拍（每 N 拍回 1）。在 `GameManager.OnBeat` 中驱动：

```csharp
// 在 GameManager 中
private void AdvanceOneBeat()
{
    Beat++; ...
    OnBeat?.Invoke(Bar, Beat);
    EnergyManager.Instance.OnGameBeat(Bar, Beat);  // 新增
    ...
}
```

## 十、完整操作流程

### 开发部署步骤

```
1. 创建 Deploy/ 文件夹，新建 4 个脚本（EnergyManager / GridHelper / DeploySystem / DragHandler）
2. 创建 DeployPanel.cs 放在 UI/ 文件夹
3. 新建 ghost 预制体（透明 Sprite）
4. 挂载组件到场景（参考上面 Hierarchy）
5. 配置 GridHelper 的 tileSize / gridWidth / gridOrigin
6. 在 DeployPanel 中填好 _availableUnits 数组
7. 运行测试：点击卡片 → 拖动 → 松开放置
```

### 用户操作流程

```
1. 游戏进入 Deploy 阶段
2. 屏幕底部显示可用单位卡片
3. 点击卡片 → 鼠标变成该单位的幽灵
4. 移入战场 → 幽灵吸附到网格
5. 绿色可放，红色不可放
6. 松开左键 → 单位部署成功，扣能量
7. 继续部署下一个
8. 也可以按右键取消当前拖拽
9. 点击"开战" → 进入 Battle，面板隐藏
```
