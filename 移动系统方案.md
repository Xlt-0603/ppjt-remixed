# 砰砰军团 — 移动系统方案（当前版本）

> 最后更新：2026-05-19

---

## 一、整体架构

移动系统分为**两层**：

```
逻辑层（UnitBase）
  └─ OnBeat 中更新 TargetPosition（目标位置，每拍跳变）
        ↓
视觉层（UnitVisual）
  └─ Update 中 SmoothDamp 把 transform.position 追向 TargetPosition
```

### 核心文件

| 文件 | 角色 |
|------|------|
| `Units/UnitBase.cs` | 所有单位的基类，定义移动逻辑 |
| `Units/EnemyBase.cs` | 敌人单位，继承 UnitBase |
| `Units/Tower.cs` | 防御塔，继承 UnitBase（不移动） |
| `Units/UnitVisual.cs` | 视觉层，处理位置的平滑追踪和朝向 |
| `Core/GameManager.cs` | 节拍引擎，驱动 OnBeat |

---

## 二、逻辑层：UnitBase

### 关键属性

```csharp
public Vector3 TargetPosition { get; set; }  // 逻辑位置，用于战斗检测和视觉跟随
```

### 移动触发时机

```
GameManager.Update()
  → beat 计时
  → AdvanceOneBeat()
  → 遍历所有单位 OnBeat(bar, beat)
       ↓
  unit.OnBeat() 中：
    ├─ 有目标 & 在攻击范围 → 攻击（不移动）
    ├─ 有目标 & 不在范围 & beat == 1 → MoveOneStepTowards(目标位置)
    ├─ 无目标 & beat == 1 → 朝默认方向移动 data.moveSpeed
    └─ beat != 1 → Idle
```

### 移动频率

**每 8 拍（一小节）移动一次**，只在 `beat == 1` 时触发。

- BPM=60 时：每 1 秒移动一次
- BPM=120 时：每 0.5 秒移动一次

### 移动距离

```csharp
TargetPosition += direction * data.moveSpeed;
```

- `data.moveSpeed` 是 `UnitData` 中的字段，默认 `0.15f`
- 每次移动增加 `moveSpeed` 个 Unity 单位的距离
- 方向只有水平（X轴）：朝目标 X 方向，或 `defaultMoveDirection.x`

### 敌方移动

`EnemyBase` 与 `UnitBase` 逻辑相同，但默认移动方向为 `Vector2.left`（向左），通过 `_moveDirection` 字段控制。

### 索敌与攻击距离

```csharp
InAttackRange(target):
  distance = |TargetPosition - target.TargetPosition|
  return distance <= data.attackRange

FindNearestEnemy():
  遍历所有存活单位
  → 跳过己方和死亡单位
  → 找出 TargetPosition 距离最近的敌方单位
```

## 三、视觉层：UnitVisual

### 位置追踪

用 `SmoothDamp` 实现带加减速的平滑跟随：

```csharp
transform.position = Vector3.SmoothDamp(
    transform.position,        // 当前位置
    _unit.TargetPosition,      // 目标位置（逻辑位置）
    ref _velocity,             // 当前速度（自动维持）
    _smoothTime                // 平滑时间
);
```

### 平滑时间

| `_smoothTime` 值 | 效果 | 适用场景 |
|---|---|---|
| 0.01 | 几乎瞬移，极快跟随 | 需要即时响应 |
| **0.04**（默认） | ~2-3帧追到，带轻微加减速 | 推荐，兼顾响应和顺滑 |
| 0.08 | ~5帧追到，明显缓动 | 强调移动感 |
| 0.15 | ~9帧追到，缓慢滑动 | 缓慢浮游类单位 |

### 人物朝向

每一帧根据 `TargetPosition.x - transform.position.x` 判断：
- 差值为正（目标在右）→ `flipX = false`
- 差值为负（目标在左）→ `flipX = true`
- 差值过小（≈0）→ 不翻转

## 四、节拍引擎（GameManager）

```
BPM = 60  →  每秒 1 小节  →  每拍 0.125s
BPM = 120 →  每秒 2 小节  →  每拍 0.0625s
```

**节拍循环**：
```
Update():
  beatTimer += Time.deltaTime
  beatInterval = 60 / (BPM * 8)
  while (beatTimer >= beatInterval):
    beatTimer -= beatInterval
    AdvanceOneBeat()

AdvanceOneBeat():
  Beat++ (1~8)
  if Beat > 8: Beat = 1, Bar++
  广播 OnBeat 事件
  遍历所有单位: unit.OnBeat(bar, beat)
  胜负判定
```

## 五、UnitData 移动相关字段

| 字段 | 默认值 | 含义 |
|------|--------|------|
| `moveSpeed` | 0.15 | 每次移动的距离（Unity 单位） |
| `attackRange` | 1.0 | 攻击距离（Unity 单位） |

> **注意**：这两个值目前**没有做任何换算**。单位在 Inspector 填多少就是多少 Unity 单位。

## 六、需要讨论的点

### 6.1 TileSize / 格的概念

问题：一格应该对应多少 Unity 单位？
- 用户测试 80px / 60PPU = 1.333 不对
- 用户测试 moveSpeed=1.13 时正好是一格
- 但又有疑问：80px 是 1.333，为什么测试是 1.13？

**待决定**：需不需要引入格的概念让用户写 1 代表一格？

### 6.2 移动频率

当前：每 8 拍（一小节）才移动一次
- 设计文档说"每拍移动 1 格"
- 但用户代码一直用的是 beat==1

**待决定**：应该每拍移动还是每小节移动一次？

### 6.3 水平移动限制

当前 `MoveOneStepTowards` 用的是归一化方向向量：
```csharp
Vector3 dir = (target - TargetPosition).normalized;
TargetPosition += dir * moveSpeed;
```

这允许对角移动。是否要限制为**纯水平移动**（只改 X 轴）？

## 七、移动流程完整时序

```
Frame N:
  1. GameManager.Update()
     → beat 到达 → AdvanceOneBeat() → unit.OnBeat()
       → beat==1 → TargetPosition += direction * moveSpeed
  2. UnitVisual.Update()
     → SmoothDamp(transform.position → TargetPosition)

Frame N+1:
  1. GameManager.Update()
     → 继续计时
  2. UnitVisual.Update()
     → 继续向 TargetPosition 平滑移动

...重复直到 TargetPosition 追上 → 视觉归位
```
