## 5. 拖拽放置系统

**方案文档：** `PROJECT_PLAN.md`

### 核心文件

| 文件 | 说明 |
|------|------|
| `Assets/scripts/Deploy/DeploySystem.cs` | 费用管理 + 部署验证 + 单位生成 + 待部署队列 |
| `Assets/scripts/Deploy/DeployUI.cs` | OnGUI 卡牌面板 + 拖拽处理 + 幽灵显示 |

### 卡牌面板（DeployUI）
- 底部 OnGUI 面板，位置大小全 Inspector 可调
- 卡牌配置：`DeployCardEntry`（`UnitData` + 自定义 `CardImage`）
- 费用不足时灰色，可部署时绿色边框
- 点击卡片开始拖拽

### 拖拽流程
1. 点击卡片开始拖拽
2. 幽灵（SpriteRenderer）跟随鼠标，吸附到最近格子
3. 绿色 = 可部署，红色 = 不可部署
4. 松开鼠标 → 入队待部署（格子预留）
5. 右键取消

### 待部署系统（踩坑记录）
- **v1**：`SkipNextMove` → 新单位等一整小节 → 用户嫌太久 + 穿模
- **v2**：`LateUpdate` 延迟执行 → 无穿模但后方单位多等一小节
- **v3（最终）**：`OnBeat` 执行部署 + 手动注册/占位（不等 `Start` 延迟） + 右到左级联处理 → 三人同时前进无空格无穿模

### 关键修复
- 新单位生成的 `_gridPos` 被 `Start()` 用 `WorldToGrid(transform.position)` 覆盖 → 加 `GridInitialized` 标记跳过
- Reserve 机制从 `CanOccupy` 中移除 → 只防重复部署，不拦单位移动
- 部署范围动态计算：最远列 = min(最前我方单位-1, 最左敌方单位-1)

### Tower 修正
- `Tower.OccupiedCols` 从 1 改为 0 → 塔不占格子
